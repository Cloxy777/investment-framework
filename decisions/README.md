# Decisions Log

Dated records of actual actions taken (buys, trims, exits, rebalances) and *why* — distinct from [session logs](../sessions/), which capture the analysis. A decision entry should be short: what you did, the session log it came from, and the thesis/catalyst/timeline per Rule 10 of the [fair value methodology](../framework/fair-value-methodology.md).

Name files `YYYY-MM-DD-<action>-<ticker>.md`, e.g. `2026-06-07-trim-aapl.md`.

This is also where framework changes themselves should be logged — e.g. when a new hybrid upgrade is adopted or a threshold is revised — so you can trace *why* the framework looks the way it does at any point in time.
