# Session Logs

Dated records of analysis sessions (screenings, re-scores, new-position evaluations, rebalances). One file per session, named `YYYY-MM-DD-<task-type>-<subject>.md`, e.g. `2026-06-07-rescore-msft.md`.

Keeping these lets you:
- Audit your own valuation calls over time (see Rule 10 in [fair-value-methodology.md](../framework/fair-value-methodology.md))
- Avoid re-deriving the same fair-value model from scratch each quarter
- Track when each holding was last reviewed (feeds [holdings.md](../portfolio/holdings.md))

A session log should capture, at minimum: date, task type, tickers covered, the full score calculation, the resulting action, and the next review trigger — i.e. the output format defined in the [operating brief](../framework/operating-brief.md).
